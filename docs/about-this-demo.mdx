## What this is

This site is a **fan-made, unofficial demo**. It exists to answer one question concretely: *what does it take to give a Salesforce open-source project a modern, searchable, AI-ready docs site?* The answer, it turns out, is a Markdown folder and one CLI tool — [Blume](https://useblume.dev).

It is **not** affiliated with, endorsed by, or produced by the maintainers of Nebula Logger.

## What this isn't

This is not a replacement for the official documentation. For anything authoritative — installation, releases, issues, the full wiki — go straight to the source:

## How this was built

1. Pulled the real Nebula Logger README and public docs as raw source material
2. Reorganized it around three audiences — developers, admins, and architects — instead of one long linear page, since that's how Salesforce teams actually divide up this kind of work
3. Ran `npx blume init`, dropped the content in as `.mdx` files, and used Blume's built-in `Tabs`, `CodeGroup`, `CardGroup`, and `Panel` components with zero custom components written
4. Built and deployed as a static site — no server, no CMS, no ongoing maintenance beyond editing Markdown
