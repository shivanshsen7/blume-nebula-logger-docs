**Nebula Logger** is a 100%-native, open-source observability framework for Salesforce — logging for Apex, Lightning Web Components, Aura, Flow, Process Builder, and OmniStudio, all from one unified data model. It's MIT-licensed and one of the most widely adopted developer tools in the Salesforce ecosystem.

<GithubInfo owner="jongpie" repo="NebulaLogger" />

This site is **not** the official Nebula Logger docs. It's a demo: the same real, open-source content, rebuilt with [Blume](https://useblume.dev) — a zero-config, AI-ready documentation framework — to show what a modern docs experience can look like for a Salesforce package, with almost no extra engineering effort. See [About this demo](/about-this-demo) for the full story and credit to the original project.

## Why this matters

Salesforce teams ship a lot of internal tooling, packages, and frameworks — but documentation is usually the first thing to fall behind. This demo exists to make one point concretely: turning a well-written README into a real, searchable, AI-ready docs site doesn't have to be a project. It can be an afternoon.

<Panel>
  Try the search (`Ctrl K`), toggle dark mode top-right, or open any page's **"Copy as Markdown"** action — all of it works out of the box, with zero custom code written for this demo.
</Panel>
